Web Application Starter Project

This App was developed in order to keep track of a NoMachine host computer with a dynamic dns running behind a firewall.
In order to do this a scheduled task needs to be set up on the desired machine to anounce its presence
and update the apps datastore. Here is a sample scheduled task on a linux machine
Code:
#!/bin/bash
#
#
#GAE servlet client ip updater
wget -qO- http://ronwright71-proxy-app.appspot.com/proxyapp/hosting/clientUpdServlet?clientName={your client name}| \
grep {your client name}|awk -F ">" '{ print $2 }'|awk -F "<" '{ print $1 }'|awk -F "=" '{ print $2 }' \
>> ~/updateProxyApp.log; date >> ~/updateProxyApp.log
Wget for Windows is needed for setting up a scheduled task to update the apps datastore.
Then a scheduled task would be setup to run the following batch file updateProxyApp
Code:
wget -qO- http://ronwright71-proxy-app.appspot.com/proxyapp/hosting/clientUpdServlet?clientName={your client name}>>c:/temp/updateProxyApp.log
The scheduled task can be set to run at whatever interval desired, currently I have been testing it
on an hourly basis this means that at most it would be unreachable for less than an hour upon dns name change.
